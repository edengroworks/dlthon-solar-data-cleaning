# dlthon-solar-data-cleaning

태양광 패널 이미지의 중복·라벨 오류·비관련 데이터를 탐색하고 정제하여,  
태양광 패널의 먼지 오염 여부를 분류하는 데이터 중심 DLthon 프로젝트입니다.

---

## 1. 프로젝트 개요

이번 프로젝트에서는 여러 경로를 통해 수집된 태양광 패널 이미지 데이터를 활용하여  
패널의 먼지 오염 여부를 분류합니다.

수집된 데이터에는 다음과 같은 데이터 품질 문제가 포함되어 있을 가능성이 있습니다.

- 동일하거나 매우 유사한 중복 이미지
- 실제 이미지 내용과 맞지 않는 라벨
- 태양광 패널과 관련 없는 이미지
- Train과 Validation 사이의 유사 이미지로 인한 데이터 누수

이러한 문제는 모델이 이미지의 실제 특징보다 특정 이미지나 패턴을 암기하게 만들고,  
검증 성능을 왜곡하거나 일반화 성능을 낮출 수 있습니다.

따라서 본 프로젝트에서는 모델 구조를 계속 변경하는 방식보다  
데이터의 품질을 점검하고 정제하는 **데이터 중심 접근법(Data-Centric Approach)**을 적용합니다.

먼저 여러 모델과 Optimizer 조합을 비교하여 기준 모델을 선정하고,  
이후 모델과 주요 학습 조건을 고정한 상태에서 데이터만 변경합니다.

정제 전후의 Validation ROC-AUC와 Kaggle 제출 결과를 비교하여  
어떤 데이터 문제가 모델 성능에 영향을 미치는지 분석하는 것을 목표로 합니다.

---

## 2. 프로젝트 목표

본 프로젝트의 주요 목표는 다음과 같습니다.

1. 태양광 패널 이미지 데이터의 분포와 특성을 파악합니다.
2. 완전 중복 및 근접 중복 이미지를 탐색합니다.
3. 라벨 오류가 의심되는 이미지를 탐색합니다.
4. 태양광 패널과 관련 없는 이미지를 탐색합니다.
5. 데이터 정제 전후의 ROC-AUC 변화를 비교합니다.
6. 데이터 정제가 모델 성능에 미치는 영향을 분석합니다.
7. 재현 가능한 데이터 정제 및 학습 파이프라인을 구축합니다.

---

## 3. 데이터 구성

### 학습 데이터

- 학습 이미지: 1,366장
- 클래스 수: 2개

### 테스트 데이터

- 테스트 이미지: 468장
- 테스트 데이터에는 정답 라벨이 제공되지 않습니다.

### 라벨

| Label | 의미 |
|---:|---|
| `0` | Clean |
| `1` | Dusty |

### 제출 형식

| 컬럼 | 설명 |
|---|---|
| `id` | 테스트 이미지 식별자 |
| `dusty_prob` | 해당 이미지가 Dusty일 확률 |

제출 파일 예시:

```csv
id,dusty_prob
test_0001,0.8321
test_0002,0.1275
````

## 4. 평가 지표

본 프로젝트의 주요 평가 지표는 **ROC-AUC**입니다.

ROC-AUC는 특정 임계값 하나에만 의존하지 않고,
모델이 Clean 이미지와 Dusty 이미지를 얼마나 잘 구분하는지를 평가합니다.

또한 제출값이 최종 클래스가 아니라 `dusty_prob` 형태의 확률값이기 때문에
확률 기반 분류 성능을 비교하기에 적합합니다.

## 5. 초기 베이스라인 비교

데이터 정제 실험에 사용할 기준 모델을 선정하기 위해
다음 모델과 Optimizer 조합을 비교했습니다.

| Model           | Optimizer | Best Validation ROC-AUC |
| --------------- | --------- | ----------------------: |
| ResNet18        | Adam      |              **0.7879** |
| ResNet18        | AdamW     |                  0.7868 |
| EfficientNet-B0 | AdamW     |                  0.7589 |
| EfficientNet-B0 | Adam      |                  0.7337 |

초기 비교 결과, 가장 높은 Validation ROC-AUC를 기록한
**ResNet18 + Adam** 조합을 데이터 정제 실험의 기준 모델로 선정했습니다.

데이터 정제 효과를 공정하게 비교하기 위해

## 8. 데이터 중심 접근

본 프로젝트에서는 다음 세 가지 데이터 문제를 중심으로 정제 실험을 진행합니다.

### 8.1 중복 이미지

다음과 같은 중복 이미지를 탐색합니다.

* 파일 내용이 완전히 동일한 이미지
* 크기만 다르게 저장된 이미지
* 밝기나 색감이 일부 변경된 이미지
* 회전 또는 좌우 반전된 이미지
* 일부 영역이 잘린 이미지
* 동일한 장면을 연속 촬영한 이미지

완전 중복 이미지는 Hash 값을 활용해 탐색하고,
근접 중복 이미지는 이미지 특징이나 유사도 기반 방법을 활용하여 탐색합니다.

특히 유사한 이미지가 Train과 Validation에 동시에 포함될 경우
데이터 누수가 발생할 수 있으므로 중복 그룹 단위 분할도 검토합니다.

---

### 8.2 라벨 오류 의심 이미지

다음과 같은 이미지를 라벨 오류 후보로 정의합니다.

* Clean 이미지처럼 보이지만 Dusty로 라벨링된 이미지
* Dusty 이미지처럼 보이지만 Clean으로 라벨링된 이미지
* 모델 예측 확률과 실제 라벨의 차이가 큰 이미지
* 여러 모델이 지속적으로 반대 클래스로 예측하는 이미지
* 중복 또는 근접 중복 이미지 사이에 서로 다른 라벨이 부여된 경우

모델의 예측만으로 라벨을 자동 변경하지 않고,
후보 이미지를 추출한 뒤 사람이 직접 검토하는 방식을 우선 적용합니다.

---

### 8.3 비관련 이미지

다음과 같은 이미지를 비관련 이미지 후보로 탐색합니다.

* 태양광 패널이 존재하지 않는 이미지
* 제품 그래픽 또는 카탈로그 이미지
* 태양광 패널보다 주변 풍경이 중심인 이미지
* 다른 물체가 이미지의 대부분을 차지하는 경우
* 지나치게 흐리거나 손상되어 판별이 어려운 이미지
* 학습 목적과 관련 없는 화면 캡처 또는 합성 이미지

비관련 이미지가 학습 데이터에 포함될 경우
모델이 태양광 패널의 먼지 여부와 관계없는 특징을 학습할 수 있습니다.

## 9. 데이터 정제 원칙

데이터 정제 과정에서는 다음 원칙을 적용합니다.

1. 원본 데이터는 직접 수정하지 않습니다.
2. 제거 및 라벨 수정 대상은 별도의 CSV 파일로 기록합니다.
3. 한 번에 하나의 데이터 문제만 변경하여 실험합니다.
4. 모델과 학습 조건은 가능한 한 동일하게 유지합니다.
5. 정제 기준과 판단 근거를 함께 기록합니다.
6. 애매한 이미지는 임의로 삭제하지 않고 `review` 상태로 분류합니다.
7. 모든 실험은 다시 실행할 수 있도록 노트북과 결과를 저장합니다.

데이터 검토 상태는 다음과 같이 관리합니다.

| 상태        | 의미          |
| --------- | ----------- |
| `keep`    | 학습 데이터에 유지  |
| `remove`  | 학습 데이터에서 제외 |
| `relabel` | 라벨 수정       |
| `review`  | 추가 검토 필요    |

## 10. 실험 방법

기준 모델과 주요 학습 조건을 고정한 뒤
데이터 조건만 변경하여 ROC-AUC 변화를 비교합니다.

| 실험           | 데이터 조건            | 모델       | 평가 지표   |
| ------------ | ----------------- | -------- | ------- |
| Baseline     | 원본 데이터            | ResNet18 | ROC-AUC |
| Experiment 1 | 완전 중복 이미지 처리      | ResNet18 | ROC-AUC |
| Experiment 2 | 근접 중복 및 데이터 누수 처리 | ResNet18 | ROC-AUC |
| Experiment 3 | 비관련 이미지 정제        | ResNet18 | ROC-AUC |
| Experiment 4 | 라벨 오류 의심 이미지 정제   | ResNet18 | ROC-AUC |
| Final        | 효과가 확인된 정제 항목 결합  | ResNet18 | ROC-AUC |

각 실험에서는 다음 내용을 기록합니다.

* 실험 목적
* 실험 가설
* 변경한 데이터 조건
* 고정한 모델 조건
* 제거 또는 수정한 데이터 수
* Validation ROC-AUC
* Kaggle 제출 점수
* 결과 해석
* 실패 원인
* 다음 실험 계획
